## About

I'm a software engineer who builds across the full delivery stack — React interfaces, REST APIs, Docker containers, and the CI/CD pipelines that tie them together. At IonIdea, I shipped a client-facing Outcome-Based Education platform: TypeScript + React frontend used by real institutions, Zod-validated API integration, and a Jenkins + Docker pipeline I co-built from scratch to take it from commit to production.

Where I stand: early in my career, with one real production cycle behind me, a clear direction toward systems and DevOps engineering, and hands-on time on both the product and the infrastructure it runs on.

How I think: I don't consider a feature done when it works locally. I consider it done when it deploys cleanly, survives an environment switch, and doesn't need someone watching it at 3 AM. That means I treat the Dockerfile and the deploy config with the same attention I give the component code. I use AI tools actively in my workflow — not to replace thinking, but to move faster through the parts that don't require it, so I can spend time on the parts that do.
